Archivo LEEME.txt
el archivo **.tar.gz contiene los siguientes archivos:

lista.h: archivo que contiene la definicion de la estructura Lista y sus
procedimientos asociados

listap.h: archivo que contiene la definicion de la estructura Listap y sus
procedimientos asociados

proc.h: archivo que contiene las funciones procpadre(), prochijo() y comprobacion()

pargrep.c: archivo que contiene el main del programa y las llamadas a procpadre()
y comprobacion()

LEEME.txt: archivo que contiene el nombre los archivos que vienen con el .tar.gz
un breve descripcion de estos, el objetivo del programa, como ejecutarlo y su estructura

makefile: archivo makefile el cual se usa para compilar el programa


El programa pargrep ejecuta una busqueda concurrente de palabras en archivos de
texto, por medio de un proceso padre que crea varios procesos hijos y le pasa una 
palabra a buscar a cada hijo, los hijos buscan la palabra asignada en el archivo 
de entrada y le envian al padre cuantas ocurrencias de las palabra asignada
se encontraron en el archivo; esto continua asi hasta que no haya mas palabras a
buscar por lo cual el padre imprime en un archivo de texto, cuantas ocurrencias
de cada palabra a buscar se encontraron; los hijos al terminar su ejecucion 
imprimen por consola su pid, el pid de su padre y cuales palabras les fue asignado
a buscar.


Ejecucion:

Para ejecutar el programa, primero se compila por medio del makefile que viene con 
este archivo luego se escribe por consola el siguiente script
pargrep { -h | [-n i] {-f archpalabras | -w palabra} entrada salida }
donde:
-h (ayuda o help). Imprime un mensaje sencillo por pantalla indicando cómo invocar 
   el programa, qué significa cada opcion.

-n i cantidad de hijos a ser creados.

-f archpalabras especifica un archivo (archpalabras) con las palabras a buscar.

-w palabra especifica una palabra a ser buscada. Si sólo se busca una palabra, se 
   puede usar esta opción y así evitar usar un archivo.

entrada: archivo de texto donde serán buscadas las palabras

salida: archivo que contendrá una línea por cada palabra buscada, junto con el 
	número de veces que se encontró

{ A | B }: indica que se tienen dos opciones para escoger A o B, no pueden ser 
	ambas pues estas son excluyentes

[-n i]: Los parámetros especificados entre corchetes son opcionales, en caso de
	que no se especifique este tendrá el valor 1.

los parámetros entrada y salida son obligatorios.

Estructura:

El programa contiene funciones para hacer su ejecucion y estructuras para manejar
los datos.

funciones:

comprobacion() es el procedimiento que verifica que los parametros de entrada sean correctos
y que no haya error al introducirlos

procpadre() es el procedimiento principal del programa, alli se crean las estructuras de datos
necesarias para la ejecucion, se crean los procesos hijos y la estructura del pipe, tambien
contiene el codigo de ejecucion del padre y de los hijos

prochijo() es la funcion principal de hijo, alli es donde cuenta la cantidad de ocurrencias
de la palabra en el archivo y verifica si la se introdujo la palabra clave "NOMAS", la cual
es la clave para indicarle al hijo que no hay mas palabras a buscar.

insertar() es el procedimiento que inserta las palabras a buscar en una lista

arpal() el es procedimiento el cual escribe en el pipe la siguiente palabra a buscar por cada hijo

liberar() procedimiento que libera la memoria de la lista

insertarp() procedimiento que inserta las palabras a buscar asociadas a cada hijo en una lista

busprt() procedimiento que busca las palabras asociadas a cada proceso hijo y las imprime por 
pantalla

liberarp() procedimiento que libera la memoria de la lista

estructuras:

En el programa se usaron ciertas estructuras especiales para la buena ejecucion del programa

Matriz de pipes: es una matriz dinamica de pipes, esta es la via de comunicacion entre el padre y 
los hijos y es por donde se envian las palabras a buscar y el numero de ocurrencias por palabra

Lista: es una estructura de tipo lista la cual contiene las palabras a buscar introducidas por un
archivo

Listap: es un estructura de tipo lista la cual contiene las palabras buscadas por cada hijo y
el pid del hijo que busca la palabra asignada
